Langfuse & AI tracing

Langfuse is purpose-built for AI applications. It does for LLM calls what Datadog APM does for microservices — shows you exactly where time and money go inside each AI request. This module covers how it works, why lazy initialization matters, and how to build the tracing abstraction correctly.

What Langfuse actually gives you

trace
One trace = one user request end-to-end. Created once per rag.answer() call. Contains all child spans. Has metadata: user_id, workspace_id, total latency.
span
A step inside a trace. retrieval, context_building, llm_generation are each spans. Spans record: start time, end time, input data, output data, errors.
generation
A special span for LLM calls. Automatically tracks model, prompt tokens, completion tokens, cost. Langfuse calculates cost from its model price list.
observation
The parent class of span and generation. Any timed unit of work inside a trace. You usually use spans for your code and let Langfuse auto-detect generations.

Why lazy initialization — the most important design decision

Never initialize Langfuse at startup. If Langfuse Cloud has a DNS issue, network timeout, or outage, your API startup fails. Your users can't chat. Observability should never block serving traffic.
# ❌ BAD: Langfuse initializes at API startup # If cloud.langfuse.com is unreachable → API won't start async def lifespan(app): setup_logging() client = Langfuse( ← network call at startup public_key=settings.LANGFUSE_PUBLIC_KEY, secret_key=settings.LANGFUSE_SECRET_KEY, ) app.state.langfuse = client ← now every request depends on this yield
# ✅ GOOD: Langfuse initializes lazily on first use # API starts instantly regardless of Langfuse status # observability/langfuse_client.py _client = None _initialized = False def get_langfuse_client(): global _client, _initialized if _initialized: return _client ← instant return after first call if not settings.langfuse_enabled: logger.warning("Langfuse disabled: keys not set") _initialized = True return None try: _client = Langfuse( public_key=settings.LANGFUSE_PUBLIC_KEY, secret_key=settings.LANGFUSE_SECRET_KEY, host=settings.LANGFUSE_HOST, ) _initialized = True return _client except Exception as e: logger.warning(f"Langfuse init failed: {e}") _initialized = True ← mark done so we don't retry forever return None ← app continues normally

The tracing abstraction — why it matters

Never import Langfuse directly in your services. If you do, every service that needs tracing becomes tightly coupled to Langfuse. When you add agent_service, memory_service, etc., you'll copy-paste Langfuse code everywhere. Instead, build one abstraction in observability/tracing.py that all services consume.
# ❌ BAD: Langfuse imported directly in rag_service.py # Future: you'll copy this into agent_service.py, memory_service.py... from langfuse import Langfuse ← tight coupling class RagService: async def answer(self, ...): lf = Langfuse(public_key=..., secret_key=...) trace = lf.trace(name="rag.answer") # what if lf is None? what if init fails? # no consistent error handling
# ✅ GOOD: observability/tracing.py provides context managers # rag_service.py, agent_service.py all use the same interface # observability/tracing.py from contextlib import asynccontextmanager from .langfuse_client import get_langfuse_client @asynccontextmanager async def rag_trace(name: str, metadata: dict): client = get_langfuse_client() if client is None: yield NoOpTrace() ← same interface, does nothing return trace = client.trace(name=name, metadata=metadata) try: yield trace finally: client.flush() ← always flush, even on exception # Usage in rag_service.py — clean and decoupled: async with rag_trace("rag.answer", {"user_id": uid}) as trace: async with rag_span(trace, "retrieval", {...}) as span: chunks = await self._retrieve(...) span.update(output={"chunks_retrieved": len(chunks)})
# The NoOp pattern — makes Langfuse truly optional # Your code never needs to check "if trace is not None" class NoOpTrace: """Same interface as Langfuse trace. Does nothing.""" def update(self, **kwargs): pass def flush(self): pass class NoOpSpan: """Same interface as Langfuse span. Does nothing.""" def update(self, **kwargs): pass def end(self, **kwargs): pass # With NoOp, your service code is identical regardless: # Langfuse enabled: trace.update({"user_id": uid}) → sends data # Langfuse disabled: trace.update({"user_id": uid}) → silently no-ops

Full implementation — what to put in rag_service.py

# ai/services/rag_service.py — tracing wired in from observability.tracing import rag_trace, rag_span ← only this import import time class RagService: async def answer(self, question, workspace_id, user_id, role, ...): async with rag_trace("rag.answer", { "workspace_id": workspace_id, "user_id": user_id, "role": role, }) as trace: # span 1: retrieval async with rag_span(trace, "retrieval", {"question": question}) as s: t0 = time.monotonic() chunks = await self._search.search(question, workspace_id, user_id, role) s.update(output={"chunks_retrieved": len(chunks), "latency_ms": int((time.monotonic()-t0)*1000)}) # span 2: context building async with rag_span(trace, "context_building", {}) as s: used, context = self._build_context(chunks) s.update(output={"chunks_used": len(used), "char_budget": TOKEN_BUDGET_PER_REQUEST}) # span 3: LLM generation async with rag_span(trace, "llm_generation", {"model": LLM_MODEL}) as s: t0 = time.monotonic() response = await litellm.acompletion(model=LLM_MODEL, ...) if hasattr(response, 'usage'): ← capture if available s.update(output={ "tokens": response.usage.total_tokens, "latency_ms": int((time.monotonic()-t0)*1000), })

Why flush() matters — the async batch problem

Langfuse batches trace data before sending it. If your process exits before the batch is flushed, you lose the trace data. In short-lived requests, you must call client.flush() at the end of each trace. This is why the finally block in rag_trace() is non-negotiable.
# Without flush — trace might not appear in Langfuse UI trace = client.trace(name="rag.answer") span = trace.span(name="retrieval") span.end() # request ends here — data may be lost in Langfuse's in-memory buffer # With flush — trace always arrives try: trace = client.trace(...) # ... do work ... finally: client.flush() ← sends the buffer immediately, blocks until done

Config wiring — what goes in settings

# config.py — Settings class additions class Settings(BaseSettings): # ... existing fields ... # Langfuse — all optional, empty string = disabled LANGFUSE_PUBLIC_KEY: str = "" LANGFUSE_SECRET_KEY: str = "" LANGFUSE_HOST: str = "https://cloud.langfuse.com" @property def langfuse_enabled(self) -> bool: return bool(self.LANGFUSE_PUBLIC_KEY and self.LANGFUSE_SECRET_KEY) # .env.example additions: # LANGFUSE_PUBLIC_KEY=pk-lf-... # LANGFUSE_SECRET_KEY=sk-lf-... # LANGFUSE_HOST=https://cloud.langfuse.com
