Core Concepts

Understand the mental model before writing a line of code

🕸️ LangGraph — State Graphs

LangGraph lets you model agent logic as a directed graph of nodes (functions) connected by edges (routing logic). State is passed between nodes as a typed dict, so you always know what's flowing through your agent.

🔄 Agent Loop

An agentic loop is: Think → Act → Observe → Repeat. The agent decides what tool to call, executes it, sees the result, and decides next steps — all driven by the LLM's output.

🗄️ RAG — Retrieval Augmented Generation

Instead of relying solely on LLM training data, RAG retrieves relevant context at query-time from a vector store (Qdrant) and injects it into the prompt — giving the LLM fresh, grounded knowledge.

📐 Embeddings

Text → dense numerical vectors. Similar meaning → similar vectors (small cosine distance). OpenAI's text-embedding-3-small creates 1536-dimension vectors we store in Qdrant's collections.

🛠️ Tool Calling

GPT-4o can decide to call functions you define. LangGraph intercepts these calls, executes the real function (e.g. a Qdrant search), and feeds results back — letting the LLM use external data mid-reasoning.

⚡ FastAPI + Async

FastAPI's async endpoints mean many concurrent requests can share a single thread. Perfect for I/O-bound agent calls that spend most of their time waiting on LLM & vector DB network calls.

Key insight: LangGraph is the orchestrator. Qdrant is the memory. OpenAI is the reasoner. FastAPI is the interface. Each does one thing excellently.

Qdrant Vector Store

Embed your documents and set up semantic search as a tool

Qdrant OpenAI Embeddings
app/vectorstore/client.py 
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams
import os

COLLECTION = os.getenv("QDRANT_COLLECTION", "research_docs")
VECTOR_SIZE = 1536  # text-embedding-3-small dimension

def get_qdrant_client() -> QdrantClient:
    return QdrantClient(url=os.getenv("QDRANT_URL", "http://localhost:6333"))

def ensure_collection(client: QdrantClient) -> None:
    """Create collection if it doesn't already exist."""
    existing = [c.name for c in client.get_collections().collections]
    if COLLECTION not in existing:
        client.create_collection(
            collection_name=COLLECTION,
            vectors_config=VectorParams(
                size=VECTOR_SIZE,
                distance=Distance.COSINE
            )
        )
        print(f"✓ Created collection: {COLLECTION}")
app/vectorstore/ingestion.py 
from openai import OpenAI
from qdrant_client.models import PointStruct
from .client import get_qdrant_client, ensure_collection, COLLECTION
import uuid, os, textwrap

openai = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
EMBED_MODEL = os.getenv("EMBEDDING_MODEL", "text-embedding-3-small")

def chunk_text(text: str, chunk_size: int = 500) -> list[str]:
    """Split text into overlapping chunks for better retrieval."""
    words = text.split()
    chunks = []
    for i in range(0, len(words), chunk_size - 50):  # 50 word overlap
        chunk = " ".join(words[i : i + chunk_size])
        chunks.append(chunk)
    return chunks

def embed_texts(texts: list[str]) -> list[list[float]]:
    resp = openai.embeddings.create(model=EMBED_MODEL, input=texts)
    return [item.embedding for item in resp.data]

def ingest_document(text: str, source: str = "unknown") -> int:
    """Chunk, embed, and upsert a document into Qdrant."""
    client = get_qdrant_client()
    ensure_collection(client)

    chunks = chunk_text(text)
    vectors = embed_texts(chunks)

    points = [
        PointStruct(
            id=str(uuid.uuid4()),
            vector=vec,
            payload={"text": chunk, "source": source}
        )
        for chunk, vec in zip(chunks, vectors)
    ]
    client.upsert(collection_name=COLLECTION, points=points)
    print(f"✓ Ingested {len(points)} chunks from '{source}'")
    return len(points)
app/vectorstore/search.py 
from .client import get_qdrant_client, COLLECTION
from .ingestion import embed_texts

def semantic_search(
    query: str,
    top_k: int = 5,
    score_threshold: float = 0.4
) -> list[dict]:
    """Retrieve top-k relevant chunks for a query."""
    client = get_qdrant_client()
    query_vec = embed_texts([query])[0]

    results = client.search(
        collection_name=COLLECTION,
        query_vector=query_vec,
        limit=top_k,
        score_threshold=score_threshold,
        with_payload=True
    )
    return [
        {"text": r.payload["text"],
         "source": r.payload.get("source", "?"),
         "score": round(r.score, 3)}
        for r in results
    ]
Why cosine distance? Text embeddings have meaningful directions but not meaningful magnitudes. Cosine similarity measures the angle between vectors, making it ideal for semantic search regardless of text length.

LangGraph Agent

Build the stateful multi-step reasoning graph that orchestrates everything

LangGraph GPT-4o
app/agent/state.py 
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph.message import add_messages

class AgentState(TypedDict):
    """Shared state passed between all nodes in the graph."""

    # The conversation — add_messages merges lists properly
    messages: Annotated[list, add_messages]

    # Retrieved context from Qdrant
    retrieved_docs: list[dict]

    # Router decision: "rag" or "direct"
    route: str

    # Final answer string
    answer: str
Why TypedDict? LangGraph validates state at each node transition. Using typed dicts catches bugs early and makes your agent's dataflow explicit.
app/agent/nodes.py 
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage
from app.vectorstore.search import semantic_search
from .state import AgentState
import os

llm = ChatOpenAI(
    model=os.getenv("LLM_MODEL", "gpt-4o-mini"),
    temperature=0
)

# ── Node 1: Router ──────────────────────────────────────────
def route_query(state: AgentState) -> AgentState:
    """Use LLM to decide if the question needs RAG or can be answered directly."""
    last_msg = state["messages"][-1].content
    system = """You are a routing assistant.
If the question needs specific factual knowledge from documents, reply: RAG
If it's a general/conversational question, reply: DIRECT
Reply with ONLY one word."""

    response = llm.invoke([
        SystemMessage(content=system),
        HumanMessage(content=last_msg)
    ])
    route = "rag" if "RAG" in response.content.upper() else "direct"
    return {**state, "route": route}

# ── Node 2: Retrieve ─────────────────────────────────────────
def retrieve(state: AgentState) -> AgentState:
    """Semantic search in Qdrant for the last user message."""
    query = state["messages"][-1].content
    docs = semantic_search(query, top_k=5)
    return {**state, "retrieved_docs": docs}

# ── Node 3: Generate (RAG) ────────────────────────────────────
def generate(state: AgentState) -> AgentState:
    """Generate an answer grounded in retrieved context."""
    docs = state.get("retrieved_docs", [])
    context = "\n\n".join(
        [f"[Source: {d['source']}]\n{d['text']}" for d in docs]
    )
    question = state["messages"][-1].content
    system = f"""Answer based ONLY on the context below.
Cite sources inline. If context is insufficient, say so.

Context:
{context}"""

    response = llm.invoke([
        SystemMessage(content=system),
        HumanMessage(content=question)
    ])
    return {**state, "answer": response.content}

# ── Node 4: Direct LLM ───────────────────────────────────────
def direct_llm(state: AgentState) -> AgentState:
    """Answer directly without retrieval for general questions."""
    response = llm.invoke(state["messages"])
    return {**state, "answer": response.content}

# ── Router Edge Function ──────────────────────────────────────
def decide_route(state: AgentState) -> str:
    return state["route"]  # "rag" or "direct"
app/agent/graph.py 
from langgraph.graph import StateGraph, END
from .state import AgentState
from .nodes import route_query, retrieve, generate, direct_llm, decide_route

def build_agent_graph():
    """Assemble and compile the research agent graph."""
    graph = StateGraph(AgentState)

    # Register nodes
    graph.add_node("route_query", route_query)
    graph.add_node("retrieve",    retrieve)
    graph.add_node("generate",    generate)
    graph.add_node("direct_llm",  direct_llm)

    # Entry point
    graph.set_entry_point("route_query")

    # Conditional edge: router decides which path
    graph.add_conditional_edges(
        "route_query",
        decide_route,
        {
            "rag":    "retrieve",
            "direct": "direct_llm"
        }
    )

    # Fixed edges: retrieve → generate → END
    graph.add_edge("retrieve",   "generate")
    graph.add_edge("generate",   END)
    graph.add_edge("direct_llm", END)

    return graph.compile()

# Singleton — compile once at startup
agent = build_agent_graph()

app/api/routes.py 
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from langchain_core.messages import HumanMessage
from app.agent.graph import agent
from app.vectorstore.ingestion import ingest_document

router = APIRouter()

# ── Request / Response Models ────────────────────────────────
class ChatRequest(BaseModel):
    question: str
    session_id: str = "default"

class ChatResponse(BaseModel):
    answer: str
    route: str
    sources: list[dict]

class IngestRequest(BaseModel):
    text: str
    source: str = "user_upload"

# ── Endpoints ────────────────────────────────────────────────
@router.post("/chat", response_model=ChatResponse)
async def chat(req: ChatRequest):
    """Ask the research agent a question."""
    try:
        initial_state = {
            "messages": [HumanMessage(content=req.question)],
            "retrieved_docs": [],
            "route": "",
            "answer": ""
        }
        result = await agent.ainvoke(initial_state)
        return ChatResponse(
            answer=result["answer"],
            route=result["route"],
            sources=result.get("retrieved_docs", [])
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

@router.post("/ingest")
async def ingest(req: IngestRequest):
    """Embed and store a document into Qdrant."""
    count = ingest_document(req.text, req.source)
    return {"status": "ok", "chunks_stored": count}

@router.get("/health")
async def health():
    return {"status": "healthy"}

Full Integration

Connect all layers end-to-end and understand what's happening at each step

LangGraph Qdrant FastAPI OpenAI

📋 What happens on a /chat request?

1

Request hits FastAPI

FastAPI validates the ChatRequest Pydantic model and creates the initial AgentState with the user's question as a HumanMessage.

2

LangGraph invokes the compiled graph

The state flows into the route_query node. The LLM is called with a system prompt to decide: does this question need external knowledge (RAG) or can it be answered from training data (DIRECT)?

3a

RAG path → retrieve node

The user's question is embedded by text-embedding-3-small. Qdrant runs a cosine similarity search and returns the top 5 most semantically relevant chunks from your documents.

3b

Direct path → direct_llm node

General knowledge questions bypass Qdrant entirely. GPT-4o answers from its parametric memory. Faster and cheaper for non-document questions.

4

generate node (RAG only)

Retrieved chunks are formatted into a context block and injected into a system prompt. GPT-4o synthesizes a grounded answer, citing the source documents inline.

5

FastAPI returns the response

The final state is mapped to a ChatResponse: the answer, the routing decision taken, and the source chunks (with scores) that informed the answer.
