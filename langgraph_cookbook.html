01

Core Concepts & Mental Model

LangGraph models your agent as a directed graph where nodes do work and edges control flow. Unlike chain-style frameworks, it gives you explicit control over loops, branches, state, and checkpoints — essential for reliable agents.

START
→
agent
LLM call
→
tools
execute
→
agent
loop
→
END

StateGraph

The core object. Takes a TypedDict schema defining what data flows through the graph. All nodes read from and write to this shared state.

Nodes

Python functions (or runnables) that take state as input and return a partial state update. This is where your logic lives.

Edges

Connections between nodes. Can be unconditional or conditional (branching logic based on state values).

Boilerplate — Minimal Agent

from langgraph.graph import StateGraph, START, END
from typing import TypedDict, Annotated
import operator

# 1. Define State Schema
class AgentState(TypedDict):
    messages: Annotated[list, operator.add]   # list that appends
    next_step: str

# 2. Define Nodes (functions)
def agent_node(state: AgentState) -> dict:
    # do work, return partial state update
    return {"messages": ["agent response here"]}

# 3. Build Graph
graph = StateGraph(AgentState)
graph.add_node("agent", agent_node)
graph.add_edge(START, "agent")
graph.add_edge("agent", END)

# 4. Compile
app = graph.compile()

# 5. Invoke
result = app.invoke({"messages": ["hello"], "next_step": ""})

State Design Patterns

State is the single most important design decision. Get it wrong and your agent will be buggy and hard to debug. State travels through every node and persists across checkpoints.

// Rule of thumb: keep state minimal but complete. Every field should answer "do I need this to make routing decisions or for a node to do its job?"

Reducer Annotations

from typing import Annotated
import operator

class State(TypedDict):
    # REPLACE (default) — last write wins
    status: str

    # APPEND — uses operator.add reducer
    messages: Annotated[list, operator.add]

    # CUSTOM REDUCER — merge dicts
    metadata: Annotated[dict, lambda x, y: {**x, **y}]

    # COUNTER — sum integers
    retry_count: Annotated[int, operator.add]

Node Signatures

# Basic node
def my_node(state: State) -> dict:
    return {"key": "value"}

# Node with config (access thread_id, metadata, etc.)

    thread_id = config["configurable"].get("thread_id")
    return {"status": "processed"}

# Async node
async def async_node(state: State) -> dict:
    result = await some_async_call(state["messages"])
    return {"messages": [result]}

Edge Types

# Unconditional edge
graph.add_edge("node_a", "node_b")

# Conditional edge — router function returns next node name
def router(state: State) -> str:
    if state["error"]:
        return "handle_error"
    elif state["tool_calls"]:
        return "tools"
    else:
        return END

graph.add_conditional_edges("agent", router)

# Conditional edge with explicit mapping
graph.add_conditional_edges(
    "agent",
    router,
    {
        "handle_error": "handle_error",
        "tools":        "tools",
        END:             END,
    }
)

# Send to multiple nodes in parallel
from langgraph.types import Send

def fan_out(state: State):
    return [Send("worker", {"task": t}) for t in state["tasks"]]

graph.add_conditional_edges("planner", fan_out)

Agent Patterns

ReAct Agent (Think → Act → Observe loop)

from langchain_openai import ChatOpenAI
from langchain_core.messages import ToolMessage
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.prebuilt import ToolNode

model = ChatOpenAI(model="gpt-4o").bind_tools(tools)

def agent(state: MessagesState):
    response = model.invoke(state["messages"])
    return {"messages": [response]}

def should_continue(state: MessagesState) -> str:
    last = state["messages"][-1]
    return "tools" if last.tool_calls else END

builder = StateGraph(MessagesState)
builder.add_node("agent", agent)
builder.add_node("tools", ToolNode(tools))
builder.add_edge(START, "agent")
builder.add_conditional_edges("agent", should_continue)
builder.add_edge("tools", "agent")   # loop back

app = builder.compile()
python

Plan-and-Execute Pattern

class PlanState(TypedDict):
    task: str
    plan: Annotated[list[str], operator.add]
    results: Annotated[list, operator.add]
    current_step: int

def planner(state):
    plan = llm.invoke(f"Create a step-by-step plan for: {state['task']}")
    return {"plan": parse_plan(plan), "current_step": 0}

def executor(state):
    step = state["plan"][state["current_step"]]
    result = execute_step(step)
    return {"results": [result], "current_step": state["current_step"] + 1}

def is_done(state) -> str:
    return END if state["current_step"] >= len(state["plan"]) else "executor"

Multi-Agent Supervisor Pattern

# Supervisor routes between specialized agents
AGENTS = ["researcher", "coder", "reviewer"]

def supervisor(state):
    prompt = supervisor_prompt + str(state["messages"])
    decision = llm.invoke(prompt)   # returns {"next": "coder"}
    return {"next": decision["next"]}

def route_to_agent(state) -> str:
    return state["next"]   # "researcher" | "coder" | "reviewer" | END

# Each agent is a subgraph compiled separately
graph.add_node("supervisor", supervisor)
for name in AGENTS:
    graph.add_node(name, agent_factory(name))
    graph.add_edge(name, "supervisor")   # all report back

graph.add_conditional_edges("supervisor", route_to_agent)

Tools Integration

Define Tools

from langchain_core.tools import tool
from pydantic import BaseModel, Field

# Simple tool with decorator
@tool
def search_web(query: str) -> str:
    """Search the web for current information."""
    return web_search_api(query)

# Typed tool with Pydantic schema
class DatabaseQueryInput(BaseModel):
    table: str = Field(description="Table name to query")
    filter: str = Field(description="SQL WHERE clause")
    limit: int = Field(default=10, description="Max rows")

@tool(args_schema=DatabaseQueryInput)
def query_db(table: str, filter: str, limit: int) -> list:
    """Query the database and return results."""
    return db.execute(f"SELECT * FROM {table} WHERE {filter} LIMIT {limit}")

ToolNode — Automatic Execution

from langgraph.prebuilt import ToolNode

tools = [search_web, query_db, write_file]

# ToolNode auto-executes all tool_calls in last message
tool_node = ToolNode(tools)

# Add to graph
graph.add_node("tools", tool_node)

# Manual tool execution (for custom error handling)
def execute_tools(state: State):
    tool_calls = state["messages"][-1].tool_calls
    results = []
    for call in tool_calls:
        try:
            tool_fn = tool_map[call["name"]]
            out = tool_fn.invoke(call["args"])
            results.append(ToolMessage(content=str(out), tool_call_id=call["id"]))
        except Exception as e:
            results.append(ToolMessage(content=f"Error: {e}", tool_call_id=call["id"]))
    return {"messages": results}
python
// Never let tool errors crash the graph. Always wrap tool execution in try/except and return error messages back into state — let the LLM decide how to handle them.

Checkpointers

from langgraph.checkpoint.memory import MemorySaver
from langgraph.checkpoint.sqlite import SqliteSaver
from langgraph.checkpoint.postgres import PostgresSaver

# Development: in-memory
memory = MemorySaver()

# Local persistence: SQLite
with SqliteSaver.from_conn_string("./checkpoints.db") as saver:
    app = graph.compile(checkpointer=saver)

# Production: Postgres
with PostgresSaver.from_conn_string(DB_URI) as saver:
    saver.setup()   # creates tables
    app = graph.compile(checkpointer=saver)

# Invoke with thread_id for session persistence
config = {"configurable": {"thread_id": "user-123-session-1"}}
result = app.invoke({"messages": [...]}, config=config)

# Resume the same thread later
result2 = app.invoke({"messages": ["follow-up question"]}, config=config)

Human-in-the-Loop

Interrupts pause execution at a node boundary and surface the current state for human review. The graph is resumed via app.invoke(None, config) after a human takes action.

Interrupt Before / After

# Compile with interrupt_before
app = graph.compile(
    checkpointer=memory,
    interrupt_before=["tools"],       # pause BEFORE executing tools
    interrupt_after=["draft_response"] # pause AFTER drafting
)

config = {"configurable": {"thread_id": "t1"}}

# First call — pauses at interrupt
app.invoke({"messages": ["query"]}, config)

# Inspect the state
state = app.get_state(config)
print(state.values)        # see what tools will be called
print(state.next)          # ['tools']

# Human approves — resume
app.invoke(None, config)

# Human edits state before resuming
app.update_state(config, {"messages": [corrected_message]})
app.invoke(None, config)
python

Dynamic Interrupt (interrupt())

from langgraph.types import interrupt

def review_node(state):
    # Pauses here and surfaces value to human
    human_decision = interrupt({
        "question": "Approve this action?",
        "proposed_action": state["action"]
    })
    # human_decision is what the human provides on resume
    if human_decision["approved"]:
        return {"status": "approved"}
    return {"status": "rejected"}

# Resume with human input
from langgraph.types import Command
app.invoke(Command(resume={"approved": True}), config)

Reliability Patterns

Retry Logic

from langgraph.utils import RetryPolicy

# Add retry policy to a node
graph.add_node(
    "flaky_api_node",
    call_api,
    retry=RetryPolicy(
        max_attempts=3,
        wait_random_min=1,
        wait_random_max=5,
        retry_on=Exception,   # or specific exception types
    )
)
python

Recursion / Loop Protection

# Set max recursion limit at compile time
app = graph.compile(checkpointer=memory)

# Or per-invocation
result = app.invoke(
    initial_state,
    config={
        "recursion_limit": 25,   # default is 25
        "configurable": {"thread_id": "t1"}
    }
)

# Gracefully handle recursion limit
from langgraph.errors import GraphRecursionError
try:
    result = app.invoke(state, config)
except GraphRecursionError:
    # Log, alert, or return partial result
    result = app.get_state(config).values
python

Streaming for Long Tasks

# Stream events as they happen
for event in app.stream(initial_state, config, stream_mode="updates"):
    for node_name, node_output in event.items():
        print(f"[{node_name}] {node_output}")

# Stream LLM token-by-token
for chunk in app.stream(state, config, stream_mode="messages"):
    print(chunk.content, end="", flush=True)

# Multiple stream modes
for event in app.stream(state, stream_mode=["updates", "debug"]):
    handle(event)

Graph Lifecycle

  • Define state schema (TypedDict)
  • Define node functions
  • Create StateGraph(State)
  • Add nodes with add_node()
  • Add edges (add_edge / add_conditional_edges)
  • Set entry with add_edge(START, ...)
  • Compile: graph.compile()
  • Invoke / stream with config

Production Checklist

  • Use Postgres checkpointer, not MemorySaver
  • Set recursion_limit per invocation
  • Add RetryPolicy to all external calls
  • Trim messages to avoid context overflow
  • Stream for long-running workflows
  • Add interrupt_before dangerous tool nodes
  • Use thread_id + user_id namespacing
  • Log checkpoint IDs for time-travel debugging

Common Gotchas

// State reducers are additive — if you use operator.add on a list, returning an empty list [] still triggers concatenation. Return None or omit the key to leave a field unchanged.
// Nodes must return a dict, not the full state. Only include keys you want to update. Missing keys are left as-is by the state merge logic.
// Always give thread_id a stable, meaningful name (e.g. f"user-{user_id}-{session_id}"). This enables proper checkpointing, history, and time-travel.
// Use LangGraph Studio locally for visual debugging — you can see the graph, inspect state at each step, and replay runs without writing test code.
