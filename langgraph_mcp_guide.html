Overview & Architecture

foundation

LangGraph is a stateful graph framework built on top of LangChain for orchestrating multi-step AI agents. Where LangChain provides chains and tools, LangGraph provides nodes, edges, and state to model complex, cyclical workflows.

⛓
LangChain

Building blocks — LLMs, prompts, retrievers, chains, tools. The foundation layer.

🕸
LangGraph

Stateful graphs on top of LangChain. Models agents as nodes + directed edges with shared state.

🔌
MCP (Model Context Protocol)

Standard protocol for connecting LLMs to external tools & resources via JSON-RPC servers.

🤖
Agent Loop

LLM → decide → call tool → observe → decide again. LangGraph models this as a cyclic graph.

LCEL — LangChain Expression Language

LCEL uses the | pipe operator to compose chains declaratively.

pythonlcel_basics.py
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 1. Define the building blocks
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("human", "{question}"),
])

parser = StrOutputParser()

# 2. Compose with | operator (LCEL)
chain = prompt | llm | parser

# 3. Invoke
result = chain.invoke({"question": "What is LangGraph?"})
print(result)

Defining Tools

pythontools.py
from langchain_core.tools import tool
from pydantic import BaseModel, Field

# Simple tool with @tool decorator
@tool
def get_weather(city: str) -> str:
    """Get current weather for a city."""
    return f"It's sunny in {city}, 25°C"

# Tool with Pydantic schema for validation
class SearchInput(BaseModel):
    query: str = Field(description="Search query")
    max_results: int = Field(default=5, description="Max results")

@tool("web_search", args_schema=SearchInput)
def web_search(query: str, max_results: int = 5) -> list:
    """Search the web for information."""
    return [f"Result {i}: {query}" for i in range(max_results)]

# Bind tools to LLM for function-calling
llm_with_tools = llm.bind_tools([get_weather, web_search])

LangGraph Core Concepts

graph mechanics

LangGraph represents an agent as a directed graph where each node performs computation and edges route the flow based on state.

The 3 Primitives

🟦
State

Shared data schema (TypedDict) that persists across nodes. Each node reads and writes to it.

⬜
Node

A Python function that takes state and returns a partial state update. Think of it as a step.

➡️
Edge

Connection between nodes. Can be static (always go to X) or conditional (branch based on state).

Minimal Graph Example

pythonminimal_graph.py
from typing import TypedDict
from langgraph.graph import StateGraph, END

# 1. Define State schema
class AgentState(TypedDict):
    messages: list
    step_count: int

# 2. Define Nodes (functions)
def greet_node(state: AgentState) -> dict:
    return {
        "messages": state["messages"] + ["Hello!"],
        "step_count": state["step_count"] + 1
    }

def respond_node(state: AgentState) -> dict:
    last = state["messages"][-1]
    return {
        "messages": state["messages"] + [f"Responding to: {last}"]
    }

# 3. Build the graph
graph = StateGraph(AgentState)
graph.add_node("greet", greet_node)
graph.add_node("respond", respond_node)

# 4. Add edges
graph.set_entry_point("greet")
graph.add_edge("greet", "respond")
graph.add_edge("respond", END)

# 5. Compile and run
app = graph.compile()
result = app.invoke({"messages": ["Hi"], "step_count": 0})
print(result)

State & TypedDict

data model

State with Annotated Reducers

By default, returning a key in a node replaces it. Use Annotated with operator.add to append instead.

pythonstate_reducers.py
from typing import TypedDict, Annotated
import operator
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph.message import add_messages

class ChatState(TypedDict):
    # add_messages = smart reducer: appends, handles updates by ID
    messages: Annotated[list[BaseMessage], add_messages]

    # operator.add = appends list items
    logs: Annotated[list[str], operator.add]

    # No annotation = REPLACE on each update
    current_step: str
    is_done: bool

# add_messages handles HumanMessage, AIMessage, ToolMessage correctly
def chat_node(state: ChatState):
    response = llm.invoke(state["messages"])
    return {
        "messages": [response],  # appended via add_messages
        "logs": ["chat_node ran"],   # appended via operator.add
        "current_step": "chatting"  # replaced
    }
💡 Tip

Always use add_messages reducer for message lists. It handles deduplication, ToolMessage correlation, and AIMessage updates by message ID automatically.

MessagesState — Shorthand

python
from langgraph.graph import MessagesState

# Plus you can extend it:

Edges & Conditional Routing

control flow
Edge TypeMethodWhen to Use
Staticadd_edge(src, dst)Always go from src → dst
Conditionaladd_conditional_edges(src, fn)Branch based on state
Entryset_entry_point(node)First node to execute
Finishset_finish_point(node)Last node, equivalent to → END

Conditional Edges — The Router Pattern

pythonconditional_routing.py
from langgraph.graph import StateGraph, END
from langchain_core.messages import AIMessage

# Router function: examines state → returns node name or END
def should_continue(state: MessagesState) -> str:
    last_message = state["messages"][-1]

    # If LLM called a tool → go to tools node
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"

    # Otherwise → we're done
    return END

# Add conditional edge with path_map
graph.add_conditional_edges(
    "agent",            # from node
    should_continue,     # router function
    {                    # path_map: return value → node name
        "tools": "tools",
        END: END
    }
)

# After tools run, loop back to agent
graph.add_edge("tools", "agent")
ℹ️ The ReAct Loop

The agent → should_continue → tools → agent pattern IS the ReAct agent loop. The LLM reasons, calls a tool, observes the result, and repeats until no more tools are needed.

MCP is a JSON-RPC 2.0 based protocol that standardizes how LLMs connect to external tools, resources, and data sources. Think of it as a USB-C standard for AI tool integrations.

Building an MCP Server (Python)

pythonmcp_server.py
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp import types
import asyncio

# Create server instance
server = Server("my-crud-server")

# In-memory "database"
items_db: dict[str, dict] = {}

# ─── List available tools ───────────────────
@server.list_tools()
async def list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="create_item",
            description="Create a new item in the database",
            inputSchema={
                "type": "object",
                "properties": {
                    "name": {"type": "string"},
                    "value": {"type": "string"}
                },
                "required": ["name", "value"]
            }
        ),
        types.Tool(
            name="read_item",
            description="Read an item by ID",
            inputSchema={
                "type": "object",
                "properties": {"item_id": {"type": "string"}},
                "required": ["item_id"]
            }
        ),
        types.Tool(name="list_items", description="List all items",
                    inputSchema={"type": "object", "properties": {}}),
        types.Tool(
            name="update_item",
            description="Update an existing item",
            inputSchema={
                "type": "object",
                "properties": {
                    "item_id": {"type": "string"},
                    "value": {"type": "string"}
                },
                "required": ["item_id", "value"]
            }
        ),
        types.Tool(
            name="delete_item",
            description="Delete an item by ID",
            inputSchema={
                "type": "object",
                "properties": {"item_id": {"type": "string"}},
                "required": ["item_id"]
            }
        ),
    ]

# ─── Handle tool calls ──────────────────────
@server.call_tool()
async def call_tool(name: str, arguments: dict) -> list[types.TextContent]:
    import uuid, json

    if name == "create_item":
        item_id = str(uuid.uuid4())[:8]
        items_db[item_id] = {"name": arguments["name"], "value": arguments["value"]}
        return [types.TextContent(type="text", text=json.dumps({"id": item_id, "status": "created"}))]

    elif name == "read_item":
        item = items_db.get(arguments["item_id"])
        if not item:
            return [types.TextContent(type="text", text="Item not found")]
        return [types.TextContent(type="text", text=json.dumps(item))]

    elif name == "list_items":
        return [types.TextContent(type="text", text=json.dumps(items_db))]

    elif name == "update_item":
        item_id = arguments["item_id"]
        if item_id not in items_db:
            return [types.TextContent(type="text", text="Item not found")]
        items_db[item_id]["value"] = arguments["value"]
        return [types.TextContent(type="text", text=json.dumps({"status": "updated"}))]

    elif name == "delete_item":
        item_id = arguments["item_id"]
        if items_db.pop(item_id, None):
            return [types.TextContent(type="text", text=json.dumps({"status": "deleted"}))]
        return [types.TextContent(type="text", text="Not found")]

# ─── Run the server ─────────────────────────
if __name__ == "__main__":
    asyncio.run(stdio_server(server))

from langchain_mcp_adapters.client import MultiServerMCPClient

import asyncio

Tools & ToolNode

execution

ToolNode — Automatic Tool Execution

ToolNode is a prebuilt node that reads tool_calls from the last AIMessage, executes each tool, and returns ToolMessage results.

pythontool_node.py
from langgraph.prebuilt import ToolNode, tools_condition
from langgraph.graph import StateGraph, MessagesState, END
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool

@tool
def calculator(expression: str) -> float:
    """Evaluate a math expression."""
    return eval(expression)

tools = [calculator]
llm = ChatOpenAI(model="gpt-4o-mini").bind_tools(tools)

def agent_node(state: MessagesState):
    return {"messages": [llm.invoke(state["messages"])]}

# ToolNode auto-handles all tool calls
tool_node = ToolNode(tools)

graph = StateGraph(MessagesState)
graph.add_node("agent", agent_node)
graph.add_node("tools", tool_node)

graph.set_entry_point("agent")

# tools_condition = prebuilt router
# → "tools" if tool_calls present, END otherwise
graph.add_conditional_edges("agent", tools_condition)
graph.add_edge("tools", "agent")  # loop back

app = graph.compile()

Memory & Checkpointing

persistence

LangGraph's checkpointing system saves full state snapshots after each node, enabling pause/resume, time-travel debugging, and multi-turn conversations.

In-Memory Checkpointer

pythonmemory.py
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()

# Compile with checkpointer
app = graph.compile(checkpointer=checkpointer)

# thread_id groups messages into a "conversation"
config = {"configurable": {"thread_id": "user-alice-session-1"}}

# Turn 1
result1 = app.invoke(
    {"messages": [("user", "My name is Alice.")]},
    config=config
)

# Turn 2 — the agent remembers "Alice"
result2 = app.invoke(
    {"messages": [("user", "What is my name?")]},
    config=config  # same thread_id = same conversation
)

Persistent Checkpointer (SQLite / Postgres)

pythonpersistent_memory.py
# pip install langgraph-checkpoint-sqlite
from langgraph.checkpoint.sqlite import SqliteSaver
import sqlite3

conn = sqlite3.connect("agent_memory.db", check_same_thread=False)
checkpointer = SqliteSaver(conn)
app = graph.compile(checkpointer=checkpointer)

# ─── Get state history ─────────────────────
config = {"configurable": {"thread_id": "alice"}}
history = list(app.get_state_history(config))
print(f"Total checkpoints: {len(history)}")

# ─── Time travel: replay from checkpoint ──
old_checkpoint = history[-2]  # 2 steps ago
app.invoke(None, config=old_checkpoint.config)  # resumes from there
⚠️ Warning

Always use a unique thread_id per conversation. Sharing thread IDs across users will mix their conversation histories!

Step 1 — database.py

pythonmcp_server/database.py
import sqlite3, uuid
from datetime import datetime

DB_PATH = "notes.db"

def init_db():
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS notes (
            id TEXT PRIMARY KEY,
            title TEXT NOT NULL,
            content TEXT NOT NULL,
            created_at TEXT NOT NULL,
            updated_at TEXT NOT NULL
        )
    """)
    conn.commit(); conn.close()

def create_note(title: str, content: str) -> dict:
    note_id = str(uuid.uuid4())
    now = datetime.utcnow().isoformat()
    conn = sqlite3.connect(DB_PATH)
    conn.execute(
        "INSERT INTO notes VALUES (?,?,?,?,?)",
        (note_id, title, content, now, now)
    )
    conn.commit(); conn.close()
    return {"id": note_id, "title": title, "content": content,
            "created_at": now, "updated_at": now}

def read_note(note_id: str) -> dict | None:
    conn = sqlite3.connect(DB_PATH)
    row = conn.execute(
        "SELECT id,title,content,created_at,updated_at FROM notes WHERE id=?",
        (note_id,)
    ).fetchone()
    conn.close()
    if not row: return None
    return dict(zip(["id","title","content","created_at","updated_at"], row))

def list_notes() -> list[dict]:
    conn = sqlite3.connect(DB_PATH)
    rows = conn.execute(
        "SELECT id,title,content,created_at,updated_at FROM notes ORDER BY created_at DESC"
    ).fetchall()
    conn.close()
    keys = ["id","title","content","created_at","updated_at"]
    return [dict(zip(keys, r)) for r in rows]

def update_note(note_id: str, title: str = None, content: str = None) -> dict | None:
    note = read_note(note_id)
    if not note: return None
    new_title = title or note["title"]
    new_content = content or note["content"]
    now = datetime.utcnow().isoformat()
    conn = sqlite3.connect(DB_PATH)
    conn.execute(
        "UPDATE notes SET title=?, content=?, updated_at=? WHERE id=?",
        (new_title, new_content, now, note_id)
    )
    conn.commit(); conn.close()
    return {**note, "title": new_title, "content": new_content, "updated_at": now}

def delete_note(note_id: str) -> bool:
    conn = sqlite3.connect(DB_PATH)
    cursor = conn.execute("DELETE FROM notes WHERE id=?", (note_id,))
    conn.commit(); conn.close()
    return cursor.rowcount > 0

Step 2 — mcp_server/server.py

pythonmcp_server/server.py
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp import types
import asyncio, json
from .database import init_db, create_note, read_note, list_notes, update_note, delete_note

init_db()
server = Server("notes-crud-server")

TOOLS = [
    types.Tool(name="create_note", description="Create a new note",
               inputSchema={"type":"object","properties":{"title":{"type":"string"},"content":{"type":"string"}},"required":["title","content"]}),
    types.Tool(name="read_note", description="Get a note by ID",
               inputSchema={"type":"object","properties":{"note_id":{"type":"string"}},"required":["note_id"]}),
    types.Tool(name="list_notes", description="List all notes",
               inputSchema={"type":"object","properties":{}}),
    types.Tool(name="update_note", description="Update title or content of a note",
               inputSchema={"type":"object","properties":{"note_id":{"type":"string"},"title":{"type":"string"},"content":{"type":"string"}},"required":["note_id"]}),
    types.Tool(name="delete_note", description="Delete a note by ID",
               inputSchema={"type":"object","properties":{"note_id":{"type":"string"}},"required":["note_id"]}),
    types.Tool(name="search_notes", description="Search notes by keyword",
               inputSchema={"type":"object","properties":{"keyword":{"type":"string"}},"required":["keyword"]}),
]

@server.list_tools()
async def list_tools(): return TOOLS

@server.call_tool()
async def call_tool(name: str, arguments: dict):
    def resp(data): return [types.TextContent(type="text", text=json.dumps(data, indent=2))]

    if   name == "create_note": return resp(create_note(arguments["title"], arguments["content"]))
    elif name == "read_note":   return resp(read_note(arguments["note_id"]) or {"error": "Not found"})
    elif name == "list_notes":  return resp(list_notes())
    elif name == "update_note": return resp(update_note(arguments["note_id"], arguments.get("title"), arguments.get("content")) or {"error": "Not found"})
    elif name == "delete_note": return resp({"deleted": delete_note(arguments["note_id"])})
    elif name == "search_notes":
        kw = arguments["keyword"].lower()
        matches = [n for n in list_notes() if kw in n["title"].lower() or kw in n["content"].lower()]
        return resp(matches)

if __name__ == "__main__":
    asyncio.run(stdio_server(server))

Step 3 — agent/graph.py

pythonagent/graph.py
from typing import Annotated
from langgraph.graph import StateGraph, MessagesState, END
from langgraph.prebuilt import ToolNode, tools_condition
from langgraph.checkpoint.memory import MemorySaver
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage

SYSTEM_PROMPT = """You are a helpful notes assistant. You can:
- CREATE notes with title and content
- READ notes by their ID
- LIST all notes
- UPDATE notes (title or content)
- DELETE notes by ID
- SEARCH notes by keyword

Always confirm operations to the user clearly."""

def build_agent(tools: list) -> object:
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0).bind_tools(tools)

    def agent_node(state: MessagesState):
        messages = [SystemMessage(content=SYSTEM_PROMPT)] + state["messages"]
        return {"messages": [llm.invoke(messages)]}

    tool_node = ToolNode(tools)

    graph = StateGraph(MessagesState)
    graph.add_node("agent", agent_node)
    graph.add_node("tools", tool_node)
    graph.set_entry_point("agent")
    graph.add_conditional_edges("agent", tools_condition)
    graph.add_edge("tools", "agent")

    checkpointer = MemorySaver()
    return graph.compile(checkpointer=checkpointer)

from dotenv import load_dotenv
from langchain_mcp_adapters.client import MultiServerMCPClient

Agent Patterns

architecture

Human-in-the-Loop

Use interrupt_before to pause execution before sensitive nodes (e.g., delete operations) and wait for user confirmation.

pythonhuman_in_loop.py
# Compile with interruption point
app = graph.compile(
    checkpointer=checkpointer,
    interrupt_before=["tools"]  # pause before tool execution
)

config = {"configurable": {"thread_id": "t1"}}
app.invoke({"messages": [("user", "Delete note abc123")]}, config)

# State is now paused. Inspect it:
state = app.get_state(config)
pending_tools = state.values["messages"][-1].tool_calls
print(f"About to call: {pending_tools}")

# Human approves → resume by passing None
confirmed = input("Confirm? (y/n): ")
if confirmed == "y":
    app.invoke(None, config)  # resumes from checkpoint
else:
    # Override state to cancel
    app.update_state(config, {"messages": [("user", "Cancel that.")]})

Multi-Agent (Supervisor Pattern)

pythonmulti_agent.py
from langgraph.graph import StateGraph, MessagesState, END

# Each specialized agent as a node
def supervisor_node(state):
    # Supervisor decides which specialist to call
    task = state["messages"][-1].content
    if "note" in task.lower():    return {"next": "notes_agent"}
    elif "search" in task.lower(): return {"next": "search_agent"}
    else:                           return {"next": END}

graph = StateGraph(MessagesState)
graph.add_node("supervisor", supervisor_node)
graph.add_node("notes_agent", notes_agent_fn)
graph.add_node("search_agent", search_agent_fn)

graph.set_entry_point("supervisor")
graph.add_conditional_edges("supervisor",
    lambda s: s.get("next", END),
    {"notes_agent": "notes_agent", "search_agent": "search_agent", END: END}
)

    stream_mode="messages"  # or "values", "updates", "events"

    if chunk[1].get("langgraph_node") == "agent":
        print(chunk[0].content, end="", flush=True)

# Stream modes:
# "values"  → full state after each node
# "updates" → only state changes per node
# "messages" → LLM tokens as they stream
# "events"  → all internal events

Graph Building Recipe

  1. Define State with TypedDict. Use Annotated[list, add_messages] for message lists.

  2. Write Node functions: def node(state: MyState) -> dict. Return only the keys you want to update.

  3. Create StateGraph(MyState) and add_node(name, fn) for each node.

  4. Set entry point with set_entry_point(), add edges with add_edge() and add_conditional_edges().

  5. compile(checkpointer=...) — add checkpointer for memory. Pass interrupt_before for human-in-the-loop.

  6. app.invoke({...}, config={"configurable": {"thread_id": "..."}}) to run.

🔑 Key Mental Model

LangGraph = State Machine where nodes are states and edges are transitions. MCP = Tool Protocol where servers expose typed functions the LLM can discover and call. Together: the agent has state, memory, and access to any external system.
