Dockerfile — FastAPI (multi-stage)
# ── Stage 1: dependency builder ──
FROM python:3.12-slim AS builder
WORKDIR /build
COPY requirements.txt .
RUN pip install --no-cache-dir \
    --prefix=/install -r requirements.txt

# ── Stage 2: slim runtime ──
FROM python:3.12-slim
WORKDIR /app
COPY --from=builder /install /usr/local
COPY . .

# Security: non-root user
RUN useradd -m -u 1001 appuser
USER appuser

EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=10s \
  CMD curl -f http://localhost:8000/health || exit 1
CMD ["uvicorn", "app.main:app", \
     "--host", "0.0.0.0", \
     "--port", "8000", \
     "--workers", "4"]
✓ Multi-stage cuts image size by ~60%. --prefix=/install isolates deps for clean copy.

Layer caching — order matters
# ❌ WRONG — copies code before deps
COPY . .
RUN pip install -r requirements.txt

# ✅ CORRECT — deps cached unless requirements.txt changes
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .

# Best: use pip-tools for deterministic builds
COPY pyproject.toml requirements.lock ./
RUN pip install --no-cache-dir -r requirements.lock
COPY src/ ./src/
💡 Each RUN, COPY, ADD is a layer. Invalidating one busts all layers below it.

🔄 CI/CD — GitHub Actions

Automated test → build → push → deploy pipeline for your stack.

Full pipeline — .github/workflows/deploy.yml
push / PR→ lint + test→ docker build→ push to ECR/GHCR→ SSH deploy→ notify 
name: CI/CD Pipeline

on:
  push:
    branches: [main, staging]
  pull_request:
    branches: [main]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
          POSTGRES_DB: testdb
        options: >-
          --health-cmd pg_isready
          --health-interval 5s
          --health-retries 5
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.12" }
      - name: Cache pip
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: pip-${{ hashFiles('requirements*.txt') }}
      - run: pip install -r requirements.txt -r requirements-dev.txt
      - run: ruff check .
      - run: mypy app/
      - run: pytest --cov=app --cov-report=xml -x
        env:
          DATABASE_URL: postgresql://test:test@localhost:5432/testdb

  build-push:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    permissions:
      contents: read
      packages: write
    outputs:
      image-tag: ${{ steps.meta.outputs.tags }}
    steps:
      - uses: actions/checkout@v4
      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=sha,prefix=sha-
            type=raw,value=latest
      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy:
    needs: build-push
    runs-on: ubuntu-latest
    steps:
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host:     ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key:      ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /opt/myapp
            docker compose pull
            docker compose up -d --remove-orphans
            docker image prune -f

🚀 Production Deployment

Zero-downtime deploys, health checks, rollbacks, and monitoring.

Zero-downtime deploy script
#!/bin/bash — deploy.sh
set -euo pipefail

IMAGE=$1  # e.g. ghcr.io/user/app:sha-abc123
echo "→ Pulling image: $IMAGE"
docker pull $IMAGE

echo "→ Running DB migrations"
docker run --rm --env-file /opt/myapp/.env \
  $IMAGE alembic upgrade head

echo "→ Blue/green swap"
export APP_IMAGE=$IMAGE
docker compose -f docker-compose.prod.yml \
  up -d --no-deps --remove-orphans api

echo "→ Health check"
for i in {1..15}; do
  STATUS=$(curl -s -o /dev/null -w "%{http_code}" \
    http://localhost:8000/health)
  [ "$STATUS" = "200" ] && { echo "✓ Healthy"; exit 0; }
  sleep 3
done
echo "✗ Health check failed — rolling back"
docker compose up -d --no-deps api  # prev image
exit 1
FastAPI health endpoint
from fastapi import FastAPI, status
from sqlalchemy import text
from neo4j import AsyncGraphDatabase

@app.get("/health", status_code=status.HTTP_200_OK)
async def health(db: AsyncSession = Depends(get_db)):
    checks = {}
    try:
        await db.execute(text("SELECT 1"))
        checks["postgres"] = "ok"
    except Exception as e:
        checks["postgres"] = f"error: {e}"
    try:
        async with neo4j_driver.session() as s:
            await s.run("RETURN 1")
        checks["neo4j"] = "ok"
    except Exception as e:
        checks["neo4j"] = f"error: {e}"
    all_ok = all(v == "ok" for v in checks.values())
    return {"status": "ok" if all_ok else "degraded",
            "checks": checks}
Alembic — DB migrations
# Setup
alembic init alembic
# alembic.ini → set sqlalchemy.url

# Generate migration from model changes
alembic revision --autogenerate -m "add user table"

# Apply migrations
alembic upgrade head

# Rollback 1 step
alembic downgrade -1

# Show current revision
alembic current

# Run in Docker before app starts
CMD sh -c "alembic upgrade head && \
    uvicorn app.main:app --host 0.0.0.0"
⚠ Run migrations before new app version starts. Never run them inside app startup — race condition with multiple replicas.
Production checklist
  1. Non-root user in Dockerfile (USER appuser)
  2. HEALTHCHECK defined in Dockerfile or compose
  3. Resource limits set (mem_limit, cpus)
  4. Secrets injected at runtime, not baked into image
  5. DB migrations run before app scales up
  6. Nginx / load balancer in front of uvicorn
  7. TLS termination + HTTPS redirect
  8. Log aggregation (stdout → journald/CloudWatch/Loki)
  9. Readiness + liveness probes configured
  10. Image tagged with git SHA, not latest
  11. Rollback tested and documented
  12. Rate limiting on public API endpoints

Async SQLAlchemy setup
from sqlalchemy.ext.asyncio import (
    create_async_engine, AsyncSession, async_sessionmaker
)
from sqlalchemy.orm import DeclarativeBase

DATABASE_URL = "postgresql+asyncpg://user:pass@db/mydb"

engine = create_async_engine(
    DATABASE_URL,
    pool_size=20,        # connections to keep open
    max_overflow=10,    # extra burst connections
    pool_timeout=30,
    pool_recycle=1800,  # recycle every 30 min
    pool_pre_ping=True, # check before use
    echo=False,
)
AsyncSessionLocal = async_sessionmaker(
    engine, expire_on_commit=False
)

class Base(DeclarativeBase): pass

# FastAPI dependency
async def get_db():
    async with AsyncSessionLocal() as session:
        yield session
Model + relationship example
from sqlalchemy import String, ForeignKey, DateTime
from sqlalchemy.orm import Mapped, mapped_column, relationship
from datetime import datetime
import uuid

class User(Base):
    __tablename__ = "users"
    id:         Mapped[uuid.UUID] = mapped_column(
                    primary_key=True, default=uuid.uuid4)
    email:      Mapped[str]  = mapped_column(String(255), unique=True)
    created_at: Mapped[datetime] = mapped_column(
                    DateTime, default=datetime.utcnow)
    documents:  Mapped[list["Document"]] = relationship(back_populates="user")

class Document(Base):
    __tablename__ = "documents"
    id:      Mapped[uuid.UUID] = mapped_column(primary_key=True)
    user_id: Mapped[uuid.UUID] = mapped_column(ForeignKey("users.id"))
    content: Mapped[str]
    user:    Mapped["User"] = relationship(back_populates="documents")

Essential Cypher patterns
// CREATE node
CREATE (u:User {id: $id, email: $email, createdAt: datetime()})

// MERGE (upsert)
MERGE (u:User {id: $id})
ON CREATE SET u.email = $email, u.createdAt = datetime()
ON MATCH  SET u.updatedAt = datetime()

// Create relationship
MATCH (u:User {id: $userId}), (d:Document {id: $docId})
MERGE (u)-[:AUTHORED {at: datetime()}]->(d)

// Traverse 2 hops
MATCH (u:User {id: $id})-[:KNOWS*1..2]->(friend)
RETURN DISTINCT friend.email

// Shortest path
MATCH p=shortestPath(
  (a:User {id: $a})-[*]-(b:User {id: $b})
)
RETURN p, length(p)
Async query execution
async def find_related(
    session, user_id: str, depth: int = 2
) -> list[dict]:
    result = await session.run(
        """
        MATCH (u:User {id: $uid})
              -[:KNOWS*1..$depth]->(friend:User)
        RETURN friend.id AS id,
               friend.email AS email,
               count(*) AS connections
        ORDER BY connections DESC
        LIMIT 20
        """,
        uid=user_id, depth=depth
    )
    return [dict(r) async for r in result]

# Write transaction (for consistency)
async def create_user_tx(tx, data: dict):
    await tx.run(
        "MERGE (u:User {id: $id}) SET u += $props",
        id=data["id"], props=data
    )

await session.execute_write(create_user_tx, data)

Agent with tools
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain_core.tools import tool

@tool
async def search_docs(query: str) -> str:
    """Search the document knowledge base."""
    docs = await retriever.ainvoke(query)
    return "\n\n".join(d.page_content for d in docs)

@tool
async def query_graph(cypher: str) -> str:
    """Run a Cypher query against Neo4j."""
    result = await neo4j_session.run(cypher)
    records = [dict(r) async for r in result]
    return str(records)

tools = [search_docs, query_graph]
agent = create_openai_tools_agent(llm, tools, prompt)
executor = AgentExecutor(
    agent=agent, tools=tools,

AWS Secrets Manager — prod pattern
import boto3, json
from functools import lru_cache

@lru_cache
def get_secret(secret_name: str) -> dict:
    client = boto3.client(
        "secretsmanager", region_name="us-east-1"
    )
    resp = client.get_secret_value(SecretId=secret_name)
    return json.loads(resp["SecretString"])

# Usage in Settings
class ProdSettings(BaseSettings):
    @classmethod
    def from_aws(cls):
        secrets = get_secret("myapp/production")
        return cls(**secrets)

# Read secret files in Python
def read_secret(env_var: str) -> str:
    file_path = os.getenv(f"{env_var}_FILE")
    if file_path and os.path.exists(file_path):
        with open(file_path) as f:
            return f.read().strip()
    return os.getenv(env_var, "")
💡 Docker secrets mount files at /run/secrets/ — never exposed as env vars, so they won't appear in docker inspect.
