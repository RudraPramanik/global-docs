What is Qdrant?
High-performance vector database built for AI applications
⚡ Purpose-Built
Written in Rust for maximum performance. Handles billions of vectors with millisecond latency. HNSW indexing for approximate nearest neighbor search.
🔍 Hybrid Search
Combines dense vectors, sparse vectors (BM25/SPLADE), and full-text search in a single query. Perfect for RAG and semantic retrieval.
🤖 Agentic Ready
Payload filtering, named vectors, scroll API, and persistent storage make Qdrant ideal for memory-augmented AI agents.

When to use Qdrant

✅ Great for
RAG pipelines, semantic search, agent long-term memory, recommendation engines, duplicate detection, multi-modal retrieval, real-time personalization
❌ Not ideal for
Exact key-value lookups, ACID transactions, complex relational joins, or real-time OLTP transactional workloads
⚡ Sweet spot
Collections with 10K–1B+ vectors needing sub-10ms search with rich metadata filtering and optional re-ranking

client.create_collection(
    collection_name="my_docs",
    vectors_config=VectorParams(

        distance=Distance.COSINE  # COSINE | EUCLID | DOT | MANHATTAN

)

Distance Metrics

MetricBest ForNotes
COSINEText embeddings (OpenAI, Cohere, sentence-transformers)Most common for NLP. Normalizes magnitude differences.
DOTDense retrieval models, ColBERT, bi-encodersFaster if vectors are pre-normalized (= COSINE).
EUCLIDImage embeddings, geometric/spatial dataL2 distance. Sensitive to scale differences.
MANHATTANSparse feature vectorsL1 distance. More robust to outliers than Euclid.

Filtering
Narrow search results with payload conditions
⚡
Qdrant filters run during vector search (not post-filter), so they don't sacrifice performance. Always index fields used in filters for best speed.

Filter Operators

OperatorClassDescription
mustFilter(must=[...])All conditions must match (AND)
shouldFilter(should=[...])At least one must match (OR)
must_notFilter(must_not=[...])None must match (NOT)
MatchValuematch=MatchValue(value=x)Exact equality
MatchAnymatch=MatchAny(any=[...])Value in list (IN)
MatchExceptmatch=MatchExcept(except=[...])Value NOT in list
MatchTextmatch=MatchText(text=x)Full-text substring match
Rangerange=Range(gt=, lt=, gte=, lte=)Numeric / datetime range
HasIdHasIdCondition(has_id=[...])Filter by point IDs
IsEmptyIsEmptyCondition(is_empty=...)Field is null/missing
IsNullIsNullCondition(is_null=...)Field is explicitly null
NestedNested(key=, filter=)Filter inside nested arrays
GeoRadiusgeo_radius=GeoRadius(...)Geospatial circle filter
GeoBoundingBoxgeo_bounding_box=...Geospatial rectangle

Filter Examples

python
from qdrant_client.models import (
    Filter, FieldCondition, MatchValue, MatchAny,
    MatchText, Range, HasIdCondition, IsEmptyCondition,
    GeoRadius, GeoPoint, Nested
)

# Exact match
f1 = Filter(must=[
    FieldCondition(key="status", match=MatchValue(value="active"))
])

# IN list
f2 = Filter(must=[
    FieldCondition(key="category", match=MatchAny(any=["tech", "ai"]))
])

# Numeric range
f3 = Filter(must=[
    FieldCondition(key="score", range=Range(gte=4.0, lte=5.0))
])

# Full-text search
f4 = Filter(must=[
    FieldCondition(key="content", match=MatchText(text="machine learning"))
])

# Complex AND + OR
f5 = Filter(
    must=[
        FieldCondition(key="published", match=MatchValue(value=True))
    ],
    should=[
        FieldCondition(key="type", match=MatchValue(value="article")),
        FieldCondition(key="type", match=MatchValue(value="blog")),
    ],
    must_not=[
        FieldCondition(key="spam", match=MatchValue(value=True))
    ]
)

# Geospatial filter
f6 = Filter(must=[
    FieldCondition(
        key="location",
        geo_radius=GeoRadius(
            center=GeoPoint(lon=-73.9857, lat=40.7484),
            radius=5000.0  # meters
        )
    )
])

# Nested array filter
f7 = Filter(must=[
    Nested(
        key="comments",
        filter=Filter(must=[
            FieldCondition(key="user", match=MatchValue(value="alice"))
        ])
    )
])

Agentic AI Patterns
Memory, RAG, and multi-agent architectures with Qdrant

Pattern 1 — Agent Long-Term Memory

🧠 Episodic + Semantic Memory Store
Store every agent interaction as a vector point. Retrieve relevant past context before generating each response. Implements "what has this user/agent done before?"
python
import uuid
from datetime import datetime

class AgentMemory:
    def __init__(self, client, collection="agent_memory", embedder=None):
        self.client = client
        self.collection = collection
        self.embedder = embedder
        self._ensure_collection()

    def _ensure_collection(self):
        if not self.client.collection_exists(self.collection):
            self.client.create_collection(
                self.collection,
                vectors_config=VectorParams(size=1536, distance=Distance.COSINE)
            )
            # Index for fast filtering by agent/user
            self.client.create_payload_index(
                self.collection, "agent_id", PayloadSchemaType.KEYWORD)
            self.client.create_payload_index(
                self.collection, "user_id", PayloadSchemaType.KEYWORD)
            self.client.create_payload_index(
                self.collection, "memory_type", PayloadSchemaType.KEYWORD)

    def save(self, text, agent_id, user_id, memory_type="episodic", metadata=None):
        vec = self.embedder.embed(text)
        self.client.upsert(self.collection, points=[
            PointStruct(
                id=str(uuid.uuid4()),
                vector=vec,
                payload={
                    "text": text,
                    "agent_id": agent_id,
                    "user_id": user_id,
                    "memory_type": memory_type,
                    "timestamp": datetime.utcnow().isoformat(),
                    **(metadata or {})
                }
            )
        ])

    def recall(self, query, agent_id, user_id, k=5):
        vec = self.embedder.embed(query)
        return self.client.search(
            self.collection,
            query_vector=vec,
            query_filter=Filter(must=[
                FieldCondition("agent_id", match=MatchValue(value=agent_id)),
                FieldCondition("user_id", match=MatchValue(value=user_id))
            ]),
            limit=k, with_payload=True
        )

Pattern 2 — Production RAG Pipeline

📚 Retrieval-Augmented Generation
Full RAG pipeline: chunk documents → embed → store → retrieve → augment LLM prompt with top-k context.
python
class RAGPipeline:
    def __init__(self, qdrant_client, openai_client, collection="rag_docs"):
        self.q = qdrant_client
        self.oai = openai_client
        self.col = collection

    def ingest(self, documents: list[dict]):
        """Chunk, embed, and store documents."""
        points = []
        for doc in documents:
            chunks = self._chunk(doc["text"], size=512, overlap=64)
            vecs = self._embed_batch(chunks)
            for i, (chunk, vec) in enumerate(zip(chunks, vecs)):
                points.append(PointStruct(
                    id=str(uuid.uuid4()),
                    vector=vec,
                    payload={
                        "text": chunk,
                        "doc_id": doc["id"],
                        "source": doc.get("source"),
                        "chunk_index": i,
                    }
                ))
        self.q.upsert(self.col, points=points)

    def query(self, question: str, k=5, filter_dict: dict=None):
        """Retrieve + generate answer."""
        vec = self._embed(question)
        results = self.q.search(
            self.col, query_vector=vec, limit=k,
            query_filter=self._build_filter(filter_dict),
            with_payload=True
        )
        context = "\n\n".join([r.payload["text"] for r in results])
        answer = self.oai.chat.completions.create(
            model="gpt-4o-mini",
            messages=[
                {"role": "system", "content":
                    f"Answer based on context:\n\n{context}"},
                {"role": "user", "content": question}
            ]
        )
        return {
            "answer": answer.choices[0].message.content,
            "sources": [r.payload.get("source") for r in results],
            "contexts": [r.payload["text"] for r in results]
        }

    """Dynamically pick relevant tools for the LLM."""

Pattern 5 — Deduplication & Novelty Detection

python
def is_duplicate(text, threshold=0.95) -> bool:
    """Check if content already exists in the store."""
    vec = embed(text)
    results = client.search(
        "my_docs", query_vector=vec, limit=1,
        score_threshold=threshold, with_payload=False
    )
    return len(results) > 0

def smart_upsert(text, payload):
    if not is_duplicate(text):
        vec = embed(text)
        client.upsert("my_docs", points=[
            PointStruct(id=str(uuid.uuid4()), vector=vec, payload=payload)
        ])
        return True
    return False

Production Checklist

✅ Indexing
Index every payload field used in filters. Use KEYWORD for categories, FLOAT for scores, TEXT for full-text.
✅ Quantization
Enable INT8 scalar quantization for collections >1M vectors. Saves 4x RAM with minimal recall loss.
✅ Batching
Upsert in batches of 64–256 points. Use upload_collection() for millions of vectors.
⚠️ HNSW Tuning
Default m=16, ef_construct=100 works well. Increase ef at search time for better recall, not at build time.
⚠️ Async Client
Use AsyncQdrantClient for FastAPI or async agent frameworks to avoid blocking the event loop.
❌ Avoid
Don't use exact=True in production search — it bypasses HNSW and does brute force O(n) scan.
